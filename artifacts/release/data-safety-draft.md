# Google Play Data Safety 입력 초안

이 문서는 Play Console의 Data safety, Ads declaration, 권한 설명을 작성할 때 참고하는 초안입니다.
실제 입력 전 Play Console 화면의 최신 질문 문구와 맞춰 다시 확인하세요.

## 데이터 수집/공유 요약

- 자체 계정 생성: 없음
- 자체 서버로 저장 번호 업로드: 없음
- 클라우드 동기화: 없음
- 앱 내부 저장: 저장 번호 세트, 즐겨찾기, 묶음 이름, QR 보기 설정, 광고형 기능 사용 횟수, 알림 설정
- 회차 데이터 캐시: 공식 결과 조회 후 기기 내부에 캐시
- 광고 SDK: Google Mobile Ads SDK 사용

## 앱이 처리하는 데이터

| 항목 | 처리 방식 | 목적 |
| --- | --- | --- |
| 저장 번호 세트 | 기기 내부 저장 | 사용자가 저장한 번호 기록과 분석 표시 |
| 즐겨찾기/묶음 이름 | 기기 내부 저장 | 메인 화면 표시와 저장 번호 관리 |
| QR 보기 설정 | 기기 내부 저장 | 앱 내부 확인용 QR 표시 방식 유지 |
| 광고형 기능 사용 횟수 | 기기 내부 저장 | 하루 사용 제한과 중복 노출 방지 |
| 알림 설정 | 기기 내부 저장 | 사용자가 켠 로컬 알림 예약 |
| 회차 결과 캐시 | 기기 내부 저장 | 오프라인/재방문 시 빠른 표시 |
| 광고 식별자 등 광고 관련 정보 | Google Mobile Ads SDK 처리 | 광고 제공, 부정 사용 방지, 성과 측정 |

## 권한 설명

- `INTERNET`: 공식 회차 결과 조회와 Google Mobile Ads SDK 테스트 광고 표시
- `POST_NOTIFICATIONS`: 사용자가 허용한 경우 로컬 알림 표시

## Play Console 선언 메모

- 앱에 광고 포함: 예
- 복권 판매 기능: 아니요
- 현금성 게임/베팅 기능: 아니요
- 계정 생성: 아니요
- 위치 정보 수집: 아니요
- 연락처/사진/파일 접근: 아니요
- 개인정보처리방침: GitHub Pages 공개 URL 필요

## 개인정보처리방침 URL 형식

GitHub Pages 설정 후 아래 형식의 URL을 Play Console에 입력합니다.

`https://<github-id>.github.io/<repo>/privacy-policy-ko.html`
